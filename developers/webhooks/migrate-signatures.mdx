<Warning>
  **V1 is being deprecated.** All endpoints must move to V2. New endpoints are V2 by default; existing V1 endpoints will be auto-migrated after a 60-day notice period.
</Warning>

Endpoints run in **V1** or **V2**.

| Mode   | Signature                                                 | Status                                        |
| ------ | --------------------------------------------------------- | --------------------------------------------- |
| **V2** | [Standard Webhooks](https://www.standardwebhooks.com) v1  | Required. Default for endpoints after 2026-05-15. |
| **V1** | Hex over `${timestamp}.${rawBody}` (ms) + `X-Pandabase-*` | Deprecated. Migrate before the cutoff.        |

Check your mode with `GET /v2/stores/:storeId/webhooks/:webhookId` (`signatureVersion`) or the dashboard.

## V1 → V2

<Steps>
  <Step title="Deploy a V2 verifier">

    <a href="https://www.standardwebhooks.com/#libraries">

  </Step>

    <strong>Your Store → Developers → Webhooks</strong>

    Open the dropdown menu and click <strong>Send test</strong> to verify
    that your endpoint is receiving and validating events correctly.
  </Step>

  <Step title="Roll back if needed">

    Go back to <strong>Your Store → Developers → Webhooks</strong>,
    click <strong>Edit</strong>, and select the previous version.
  </Step>
</Steps>

### Differences

|                         | V1                        | V2                                      |
| ----------------------- | ------------------------- | --------------------------------------- |
| Encoding                | Hex                       | `v1,<base64>` (space-separated list)    |
| Signed string           | `${timestamp}.${rawBody}` | `${Webhook-Id}.${timestamp}.${rawBody}` |
| `Webhook-Timestamp`     | Milliseconds              | Seconds                                 |
| `Webhook-Id`            | `${webhookId}/${jobId}`   | Message id (matches `payload.id`)       |
| `X-Pandabase-*` headers | Sent                      | Not sent                                |

## X-Pandabase-* → Webhook-* (V1)

V1 endpoints already send `Webhook-*` alongside the legacy headers. Switching to them adds replay protection.

| Header             | `X-Pandabase-*`              | `Webhook-*` (V1)                   |
| ------------------ | ---------------------------- | ---------------------------------- |
| Signature          | `X-Pandabase-Signature`      | `Webhook-Signature`                |
| Timestamp          | `X-Pandabase-Timestamp` (ms) | `Webhook-Timestamp` (ms)           |
| Idempotency        | `X-Pandabase-Idempotency`    | `Webhook-Id`                       |
| Signed string      | `rawBody`                    | `${timestamp}.${rawBody}`          |
| Replay protection  | None                         | Reject events older than 5 minutes |

Same secret, same hex encoding.

<CodeGroup>

```typescript Node.js
import crypto from "node:crypto";

const TOLERANCE_MS = 5 * 60 * 1000;

export function verifyV1(
  headers: Record<string, string>,
  rawBody: string,
  secret: string,
): boolean {
  const ts = headers["webhook-timestamp"];
  const sig = headers["webhook-signature"];
  if (!ts || !sig) return false;

  const expected = crypto
    .createHmac("sha256", secret)
    .update(`${ts}.${rawBody}`)
    .digest("hex");

  if (expected.length !== sig.length) return false;
  if (!crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(sig))) return false;

  return Math.abs(Date.now() - Number(ts)) <= TOLERANCE_MS;
}
```

```python Python
import hmac, hashlib, time

TOLERANCE_MS = 5 * 60 * 1000

def verify_v1(headers, raw_body: bytes, secret: str) -> bool:
    ts = headers.get("webhook-timestamp")
    sig = headers.get("webhook-signature")
    if not ts or not sig:
        return False

    expected = hmac.new(
        secret.encode(), f"{ts}.{raw_body.decode()}".encode(), hashlib.sha256
    ).hexdigest()

    if not hmac.compare_digest(expected, sig):
        return False

    return abs(int(time.time() * 1000) - int(ts)) <= TOLERANCE_MS
```

```go Go
package webhook

import (
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "strconv"
    "time"
)

const toleranceMs = 5 * 60 * 1000

func VerifyV1(headers map[string]string, rawBody []byte, secret string) bool {
    ts := headers["Webhook-Timestamp"]
    sig := headers["Webhook-Signature"]
    if ts == "" || sig == "" {
        return false
    }

    mac := hmac.New(sha256.New, []byte(secret))
    mac.Write([]byte(ts + "." + string(rawBody)))
    if !hmac.Equal([]byte(hex.EncodeToString(mac.Sum(nil))), []byte(sig)) {
        return false
    }

    tsInt, err := strconv.ParseInt(ts, 10, 64)
    if err != nil {
        return false
    }
    skew := time.Now().UnixMilli() - tsInt
    if skew < 0 {
        skew = -skew
    }
    return skew <= toleranceMs
}
```

## Pitfalls

- Hash the **raw** body, not a re-serialized JSON object.
- Use `timingSafeEqual` / `hmac.compare_digest`. Never `==`.
- Reject stale timestamps. The signature alone doesn't stop replay.
- V2 timestamps are **seconds**, V1 are **milliseconds**.
- V2 `Webhook-Signature` is a space-separated list — check each `v1,…` entry.
- Keep your server NTP-synced. >5 min drift rejects valid events.
